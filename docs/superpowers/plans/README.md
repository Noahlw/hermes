# Plans Index

This directory collects `writing-plans`-style plans for each closed wayfinder planning ticket. The orchestrator writes one plan per closed planning ticket, named:

```
YYYY-MM-DD-T-NNN-<feature-slug>.md
```

| Date | Ticket | Feature | Status |
|---|---|---|---|
| 2026-07-23 | T-? | wayfinder-to-impl-plan-workflow | meta |

Plans are static once written. Implementation lives in branches and PRs; this index is regenerated by T-024 (Compile grand map) on a monthly cadence.

## How a plan gets here

1. A wayfinder planning ticket closes with `## Rationale / ## Acceptance / ## Hand-off` sections in its close comment.
2. `orchestrator/closure_listener.py` sees the close.
3. `orchestrator/spec_extractor.py` parses the close into a `Spec`.
4. `orchestrator/impl_spawner.py` spawns a `phase:implementation` child issue.
5. `orchestrator/plan_writer.py` writes this plan.
6. `orchestrator/executor.py` dispatches a fresh agent on the impl ticket.
7. When the impl ticket closes, `orchestrator/grandmap_updater.py` appends a line to the wayfinder map (`#1`).
